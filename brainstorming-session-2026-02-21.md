### Implementation Roadmap: Code-Snippet Loot

We've decided to prioritize the **Code-Snippet Loot** feature as our high-impact goal. Here is the decision-based roadmap for implementation:

#### 1. Foundation: The Bot Infrastructure (Current State: Minimal)
*   **Path A (Simplest)**: Add a `CompanionBot` field to the user state (Zustand).
*   **Path B (Scalable)**: Create a dedicated `BotService` in the backend that handles logic execution via a safe sandbox.
*   **Decision**: Start with **Path A** to validate the fun factor before building the sandbox.

#### 2. Loot Generation: The Fragments
*   **Action**: Update `battle-logic.js` to include a `loot_snippets` array in the win-payload.
*   **Snippet Types**: 
    - `UI_Tweak` (CSS changes)
    - `Auto_Action` (Heal/Switch logic)
    - `Data_Insight` (Hidden stats)

#### 3. Execution: The "Compiler"
*   **UI**: Add a "Code Lab" tab in the World HUD.
*   **Mechanic**: Players "Slot" fragments into 3 available bot slots.

#### 4. Success Metrics
*   **Engagement**: % of players with at least 2 active snippets.
*   **Sharing**: Number of "Snippet Builds" shared in the community.
